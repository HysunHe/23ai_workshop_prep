问题：Oracle 19C升级前要注意什么事项？
解答：
•	自动升级- 利用新的自动升级实用程序，这是现在所有升级的推荐路径。该工具的完整描述，包括它是什么、在哪里获取它、文档和分步指南位于博客 -升级您的数据库

•	升级需要多长时间？- 虽然这个问题没有一个答案适合所有人，但您可以采取一些措施来帮助缩短升级时间。阅读我的数据库升级需要多长时间？ 有关内存使用情况，请参阅使用更少的内存实现更快的 Oracle 数据库升级。
•	EBS 和自动升级 -目前不建议 EBS 客户使用自动升级。请参阅验证认证和查看最佳实践选项中的互操作性说明，选择应用程序最佳实践，然后选择 EBS。
•	Exadata DB 和 GI 升级  - 如果您使用 Exadata，请参阅以下文档了解如何升级的详细信息。  在 Oracle Linux 上运行的 Exadata 数据库机的 19c 网格基础设施和数据库升级步骤（文档 ID 2542082.1）


•	性能领域创建了四个新需要考虑的事项文档，指出了已知问题以及 19c 中要遵循的一些最佳实践。这些文档包含一些更常见的问题和推荐的解决方案/解决方法以及诊断/故障排除方法。
o	避免 19c 上数据库性能问题需要考虑的事项（文档 ID 2773012.1）
o	避免 19c 上 SQL 性能问题需要考虑的事项（文档 ID 2773715.1）
o	避免 19c 上的 SQL 计划管理 (SPM) 相关问题需要考虑的事项（文档 ID 2774029.1）
o	主动避免 19C 上出现明显错误结果问题时需要考虑的事项（文档 ID 2606585.1）


•	LiveLabs – 尝试 Oracle LiveLabs，它使您可以访问工具和技术来运行各种 19c 实验室和研讨会。
•	APEX – 探索Oracle APEX，这是世界上最流行的低代码开发平台，使您能够构建可扩展、安全的企业应用程序，具有世界一流的功能，可以部署在任何地方。APEX 是 Oracle 数据库完全支持的免费功能。
•	补丁- Oracle 建议客户在升级周期内安装最新的 RU 补丁，因为它们包含最新内容以获得最佳整体体验。
•	测试投产- 测试投入生产应该是相同的硬件、相同的操作系统/版本、数据库克隆/生产副本（相同的数据、大小、数据库结构等）。Oracle 希望预生产时的数据库补丁分析、数据库运行状况检查和测试结果能够准确复制生产。在真正的预生产环境中，应用程序测试更有价值。
•	尽早解决问题- 在生产之前识别并解决预生产环境中的问题。
•	升级周期- 典型的升级周期（包括规划、预生产、应用程序测试和实际数据库升级）是一个长达数月的过程。如果涉及操作系统升级、硬件升级或应用程序或第三方软件等其他因素，客户应相应调整其期望。
•	自定义代码-大多数客户都有自定义代码，并且此代码可能需要调整/代码修订才能在新的目标数据库版本中正常运行。在少数情况下，第三方代码供应商没有在目标数据库版本下运行他们的应用程序/代码，因此目标数据库版本存在认证问题，这可能会影响时间表并导致延迟。
•	对更改的行为进行充分审查- 最新数据库版本中记录和计划的更改行为有时会被客户视为错误，但事实并非如此。请参阅此处的19c新功能。另请参阅数据库升级指南中有关行为更改、已弃用和不再支持的功能的章节。
•	删除隐藏和下划线参数-如果您从 init.ora/sp 文件中删除/注释掉“旧”参数、下划线和事件，升级通常会运行得更快。这需要在数据库升级之前完成。Oracle 的一般经验法则是在数据库升级之前删除隐藏参数，除非您的应用程序供应商或 Oracle 支持部门另有规定。还建议删除以前版本中的遗留内容并从默认值开始。数据库升级完成后，您正在“调整”数据库，或者作为强大的功能和性能测试的结果，您可能需要根据需要重新添加这些参数。
•	如果您正在考虑更改cursor_sharing参数，请务必小心-如果您将其设置为特定值，则在未经适当测试的情况下不应更改此值。这不是一个小设置。这是特定于应用程序的，如果您对cursor_sharing进行更改，Oracle的建议是在预生产中进行彻底测试。
•	时间表-确保在预生产周期中留出足够的时间来充分测试升级/迁移以及应用程序测试/分析和修复（如果发现任何回归）。几乎总是，由硬编码日期（相对于准备情况）驱动的制作事件会导致后期制作出现一些问题。
•	确保有足够的资源与升级/迁移项目保持一致-（时间、人员、服务器、测试人员等）。请特别注意应用程序测试：谁负责签署就绪情况，以及他们是否分配了必要的时间来执行此操作。
•	只在生产中进行，首先在预生产中测试过的内容-这是不言而喻的，应该作为最佳实践来遵循。
•	使用有效的升级/迁移计划开始预生产升级/迁移- 进入整个升级/迁移工作的预生产阶段，并制定预生产期间要遵循的计划（实际步骤）（测试、阶段、开发、 UAT）进入生产事件。该计划应包括 DB、GI、修补以及确保生产活动成功所需执行的任何操作。
