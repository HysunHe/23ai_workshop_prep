问题： 什么是True Cache？
解答： Oracle True Cache 是一种用于 Oracle 数据库的内存缓存方案，是具有数据一致性、自动管理的 SQL 和键值（对象或 JSON）的缓存。

整体来看，它的工作原理如下：

应用程序通过 Oracle JDBC 驱动程序手动或半自动地将查询定向到 True Cache。
True Cache 实例通过使用它为其处理的数据库应用程序服务缓存的数据来满足查询。
当发生缓存未命中时，True Cache 实例会从主数据库实例中获取块。
当 True Cache 实例首次启动时，除了在缓存未命中期间获取块之外，它还会以大块的形式获取周围的块以进行预热。
缓存块后，它会通过主数据库的重做应用日志更新，通常延迟不到一秒。这类似于 Oracle Data Guard 配置中的实时重做应用。主数据库重做块由主数据库实例上的 LGWR 进程以异步模式连续发送到 True Cache 备用重做日志文件。
对 True Cache 的查询仅返回与重做应用滞后一样新的已提交数据。
对 True Cache 实例的每个查询始终返回一致的数据。
